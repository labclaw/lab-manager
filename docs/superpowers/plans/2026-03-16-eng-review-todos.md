## Design Decisions (all agreed)

### Decision 1: Auth middleware ordering (TODO-1)
**Chosen: Merge auth + audit into single middleware.**
- Avoids FastAPI middleware reverse-registration ordering bugs
- DRY: one middleware instead of two
- Same effort as separate middleware

```
BEFORE (two separate middlewares, ordering fragile):
  Request → scans_auth → audit_user → route

AFTER (single merged middleware):
  Request → auth_and_audit_middleware → route
              ├── check session cookie
              ├── reject if unauthenticated (except /api/health, /admin/login)
              ├── set current_user from session (for audit)
              └── continue to route
```

**Implementation notes:**
- Allowlist paths that don't need auth: `/api/health`, `/admin/login`, `/admin/statics`
- Staff table is the user store — query by email, check `is_active`
- Use `itsdangerous` or FastAPI's built-in session for cookie signing
- `admin_secret_key` in config.py is the signing key (already exists)

### Decision 2: Read-only DB connection (TODO-2)
**Chosen: Internal connection in `_execute_sql()`.**
- `_execute_sql()` creates its own connection from a readonly engine
- No route signature changes, no new FastAPI dependency
- Self-contained, explicit

```
database.py:
  get_engine()          → existing read-write engine
  get_readonly_engine() → NEW, lazily created, uses DATABASE_READONLY_URL

rag.py _execute_sql():
  BEFORE: uses db session from route handler
  AFTER:  creates own connection from get_readonly_engine()
          still uses passed db session for audit logging only

docker-compose.yml init SQL:
  CREATE USER labmanager_ro WITH PASSWORD 'labmanager_ro';
  GRANT CONNECT ON DATABASE labmanager TO labmanager_ro;
  GRANT USAGE ON SCHEMA public TO labmanager_ro;
  GRANT SELECT ON ALL TABLES IN SCHEMA public TO labmanager_ro;
  ALTER DEFAULT PRIVILEGES IN SCHEMA public GRANT SELECT ON TABLES TO labmanager_ro;
```

**Fallback:** If `DATABASE_READONLY_URL` is not set, fall back to main engine + `SET TRANSACTION READ ONLY` (current behavior). Log a warning.

### Decision 4: Health endpoint Gemini check (TODO-5)
**Chosen: PG + Meilisearch only. Gemini = config check.**

```python
@app.get("/api/health")
def health():
    checks = {}
    # PostgreSQL: try SELECT 1
    checks["postgresql"] = _check_pg()      # "ok" or error message
    # Meilisearch: try client.health()
    checks["meilisearch"] = _check_meili()   # "ok" or error message
    # Gemini: config check only (no API call)
    checks["gemini"] = "ok" if get_settings().extraction_api_key else "not configured"

    all_ok = all(v == "ok" for v in checks.values())
    status_code = 200 if all_ok else 503
    return JSONResponse({"status": "ok" if all_ok else "degraded", "services": checks},
                        status_code=status_code)
```
